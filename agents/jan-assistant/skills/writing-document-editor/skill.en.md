# Writing & Document Editor

Use this skill when the user asks for help drafting, rewriting, editing, or formatting text, emails, notes, reports, or documents.

## Goal

Produce clear, well-structured text that matches the user's purpose, audience, and tone.

## Steps

1. **Clarify intent** — Purpose, audience, desired length, and required tone.
2. **Gather content** — Extract key points, facts, and constraints from the user's request or uploaded files.
3. **Outline structure** — Choose sections, headings, and logical flow.
4. **Draft** — Write in a direct, concise style aligned with the Jan Assistant tone.
5. **Edit** — Remove fluff, improve clarity, fix grammar, and ensure consistency.
6. **Format** — Apply markdown, lists, tables, or code blocks as appropriate.
7. **Summarize changes** — If editing existing text, briefly explain major revisions.

## Output format

- **Final text**: the completed draft or edited version.
- **Structure notes**: headings or section rationale (optional).
- **Change summary**: list of main edits, if applicable.

## Boundaries

- Preserve the user's intended meaning; do not rewrite into a different stance.
- Do not add unsupported claims or citations unless requested.
- Ask for clarification if the purpose or audience is ambiguous.
