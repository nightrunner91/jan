# Data Analysis Guide

Use this skill when the user provides a dataset, table, or file and asks for calculations, trends, patterns, or statistical insights.

## Goal

Analyze the data methodically and explain results in plain language.

## Steps

1. **Clarify the question** — What should the analysis answer? Define metrics and success criteria.
2. **Inspect the data** — Identify columns, types, size, missing values, and obvious errors.
3. **Clean and validate** — Handle missing or inconsistent values; note assumptions.
4. **Choose a method** — Select appropriate calculations, aggregations, or statistical tests.
5. **Execute with code** — Use the code interpreter for reproducible analysis.
6. **Visualize or tabulate** — Present key findings with charts or tables where useful.
7. **Interpret and qualify** — Explain what the results mean and what they do not mean.

## Output format

- **Question**: the analytical goal.
- **Data summary**: shape, key columns, and quality notes.
- **Method**: chosen approach and assumptions.
- **Results**: main findings with supporting numbers.
- **Visualization/table**: markdown table or described chart.
- **Caveats**: limitations, outliers, and uncertainty.
- **Next steps**: recommended further analysis if useful.

## Boundaries

- Do not extrapolate beyond the data.
- Report uncertainty and sample limitations.
- If the data is insufficient for the requested analysis, say so.
