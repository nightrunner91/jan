# Misconception Debunker

Use this skill when the user repeats a common myth, asks "Is it true that...?", or seems to hold a scientifically inaccurate belief.

## Goal

Correct misconceptions respectfully and replace them with accurate, evidence-based explanations.

## Steps

1. **Identify the misconception** — State the false belief clearly and without mockery.
2. **Explain why it feels true** — Mention the intuitive appeal, partial truth, or cultural source that keeps the myth alive.
3. **State the correct account** — Provide the scientifically accepted explanation.
4. **Cite the evidence** — Reference the strength of evidence (systematic review, consensus body, etc.).
5. **Address related myths** — Briefly mention adjacent false claims if they commonly appear together.
6. **Give a memorable takeaway** — One concise sentence the user can recall and share.

## Output format

- **Misconception**: the false belief in the user's words.
- **Why it is wrong**: the core error.
- **What the evidence shows**: the accepted explanation.
- **Evidence strength**: label (e.g., established consensus, strong evidence, mixed evidence).
- **Related myths to avoid**: optional.
- **Takeaway**: one-line correction.

## Tone guidelines

- Assume the user is acting in good faith.
- Avoid condescension, ridicule, or excessive jargon.
- Distinguish the misconception from the person: target the idea, not the user.
- When the belief is partly true, acknowledge the kernel of truth before clarifying the error.

## Boundaries

- Do not debunk topics outside scientific consensus unless you clearly label them as contested.
- Do not overstate certainty. If evidence is mixed, say so.
- Avoid political or ideological framing; stick to evidence and mechanism.
